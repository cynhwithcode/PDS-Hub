# AX 디자이너란 무엇인가
> 작성일: 2026-06-02

## 내가 정의하는 AX 디자이너
UI를 그리는 것을 넘어 시스템을 설계하고
데이터와 구조로 경험을 만드는 디자이너.
단순히 화면을 예쁘게 만드는 것이 아니라
사용자와 제품 사이의 관계를 구조화하는 역할.

## UI/UX 디자이너와의 차이
- UI/UX: 화면 단위의 경험 설계
- AX: 시스템 전체의 경험 아키텍처 설계
  - 정보 구조, 데이터 흐름, 컴포넌트 체계까지 포함

## 왜 이 방향으로 전환하려 하는가
5년간 디자인 시스템을 운영하면서
"만드는 것"보다 "구조화하는 것"에 더 흥미를 느꼈다.
PDS Hub를 만들면서 이 방향이 맞다는 것을 확인했다.

## 내가 이미 하고 있던 AX적 작업들
- 피닉스다트 DS 거버넌스 구조 설계
- 리포트 UX 개선: 데이터 위계 재설계
- PDS Hub: DS 운영 문제를 제품으로 풀기
